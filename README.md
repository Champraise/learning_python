# 🛠️ Learning Python for Data Engineering

This repository documents my hands-on journey learning Python with a specialized focus on **Data Engineering** concepts, patterns, and best practices. 

It contains structured notes, practical coding exercises, and projects geared toward building efficient, scalable, and robust data pipelines.

## 🎯 Goals

- **Master Core DE Patterns**: Focus on iterators, generators, decorators, and memory-efficient data processing.
- **Data Manipulation & Wrangling**: Deep dive into libraries like `pandas`, `Polars`, and standard library tools (`json`, `csv`, `sqlite3`).
- **Pipeline Orchestration**: Build, schedule, and monitor functional ETL/ELT pipelines.
- **Robustness & Quality**: Implement unit testing (with `pytest`), strict logging, comprehensive error handling, and type hinting (`mypy`).

## 📂 Structure

- `notes/` → Structured summaries focusing on data engineering concepts (generators, memory management, database connections).
- `exercises/` → Lab-style practice problems (API ingestion, JSON flattening, SQL interactions, custom pipeline transforms).
- `projects/` → End-to-end data pipeline projects (e.g., ETL from public APIs to local database/Parquet).

## 🚀 Learning Roadmap

- [ ] **Phase 1: Advanced Core Python**
  - Generators & Iterators (for streaming large files)
  - Decorators (for timing, logging, and retrying pipeline steps)
  - Multithreading & Multiprocessing vs. Asyncio
- [ ] **Phase 2: Data Formats & Storage**
  - Processing CSV, JSON, XML, Parquet, and Avro
  - Interacting with databases (SQLite, PostgreSQL) using standard drivers and `SQLAlchemy`
- [ ] **Phase 3: Data Transformation**
  - Vectorized operations using NumPy, Pandas, and Polars
  - Data validation schemas (Pydantic, Great Expectations)
- [ ] **Phase 4: Orchestration & Automation**
  - Building lightweight ETL pipelines
  - Scheduling & orchestrating tasks
